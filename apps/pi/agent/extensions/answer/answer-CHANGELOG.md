# Answer Extension 优化总结

## 2026-04-10: 快捷键优化 - 避免误触发

### 问题
原有的 `t` / `e` 单字母快捷键容易在输入文字时被误触发（如输入 "project", "create" 等单词）。

### 解决方案
实施了"分层 Esc"方案，优化了模式切换逻辑：

#### 快捷键变更
1. **移除**: 删除了 `t` 作为模式切换键
2. **保留**: `e` 键仅在选项选择模式下可用，用于切换到文本输入
3. **新增**: 分层 Esc 逻辑
   - 在文本输入模式（有选项时）按 `Esc` → 返回选项选择模式
   - 在选项选择模式按 `Esc` → 取消整个 Q&A 流程
   - `Ctrl+C` → 始终强制取消整个流程

#### 代码修改
- 重构了 `handleInput()` 方法中的 Esc 处理逻辑
- 在选项选择模式下单独处理 `e` 键
- 更新了 UI 提示文本，清晰说明各模式下的快捷键

#### UI 提示更新
- **选项选择模式**: `↑↓ select · 1-9 quick select · e text input · Enter confirm · Esc cancel`
- **文本输入模式（有选项）**: `Tab/Enter next · Shift+Tab prev · Shift+Enter newline · Esc back/cancel`
- **文本输入模式（无选项）**: `Tab/Enter next · Shift+Tab prev · Shift+Enter newline · Esc cancel`

#### 文档更新
- 更新了 `answer-README.md` 中的功能说明和快捷键表格
- 添加了"分层退出"特性说明

### 优势
✅ **避免误触发**: 在文本输入时不会意外切换模式  
✅ **符合直觉**: Esc 逐层退出，符合常见 UI 交互习惯  
✅ **保留便捷性**: `e` 键快速切换到文本输入（仅在需要时可用）  
✅ **紧急退出**: `Ctrl+C` 作为强制退出选项

## 主要改动

### 1. 数据结构扩展

#### ExtractedQuestion 接口
```typescript
interface ExtractedQuestion {
  question: string;
  context?: string;
  options?: string[];  // ✨ 新增：选项列表
}
```

### 2. 系统提示词增强

更新了 `SYSTEM_PROMPT` 以支持选项提取：

- 识别各种格式的选项（bullet points、"or" 分隔、编号列表等）
- 自动将选项提取到 `options` 数组
- 提供了包含选项的示例输出

**示例识别能力：**
- "Do you prefer A or B?" → `["A", "B"]`
- "Choose from: 1. X 2. Y 3. Z" → `["X", "Y", "Z"]`
- "Options:\n- React\n- Vue" → `["React", "Vue"]`

### 3. QnAComponent 类增强

#### 新增状态
```typescript
private inputMode: ('select' | 'text')[] = [];      // 每个问题的输入模式
private selectedOptionIndex: number[] = [];          // 选项选择索引
private blue = (s: string) => `\x1b[34m${s}\x1b[0m`; // 新颜色
```

#### 新增方法
```typescript
switchToTextMode()      // 从选项模式切换到文本输入
switchToSelectMode()    // 从文本输入切换到选项模式
```

#### 优化方法
- `saveCurrentAnswer()` - 根据模式保存答案（选项或文本）
- `navigateTo()` - 切换问题时加载对应的输入状态
- `handleInput()` - 全面重构，支持选项选择交互
- `render()` - 动态渲染选项列表或文本编辑器

### 4. 交互体验升级

#### 选项选择模式的新快捷键

| 快捷键 | 功能 |
|--------|------|
| `↑` `↓` | 上下选择选项 |
| `1-9` | 直接选择对应编号的选项（并自动进入下一题）|
| `Enter` | 确认当前选择并进入下一题 |
| `t` 或 `e` | 切换到文本输入模式 |
| `Tab` | 跳到下一个问题 |

| 快捷键 | 功能 |
|--------|------|
| 正常输入 | 编辑文本 |
| `Enter` | 下一题 |

| `t` 或 `e` | 如有选项，切换到选项选择模式 |
| `Tab` | 跳到下一个问题 |
| `Shift+Tab` | 跳到上一个问题 |

## 技术亮点

### 1. 智能模式切换
- 有选项的问题默认显示选项列表
- 无选项的问题自动使用文本输入
- 用户可随时在两种模式间切换（如需要自定义答案）

### 2. 答案状态保持
- 从选项模式切换到文本模式时，会预填当前选中的选项
- 从文本模式切换回选项模式时，会尝试匹配已输入的文本

### 3. 性能优化
- 渲染结果缓存（仅在状态变化时重新渲染）
- 使用数组索引管理多问题状态

### 4. 用户体验
- 数字快选：按 1-9 可秒选并自动进入下一题
- 进度可视化：圆点指示器清晰显示答题进度
- 上下文显示：问题的上下文信息以引用格式显示
- 友好提示：每个模式都显示相关的操作提示

## 已知限制

1. **数字快选仅支持 1-9**
   - 如果选项超过 9 个，超出部分只能用方向键选择
   - 考虑到实际使用场景，9 个选项已足够

2. **选项文本长度**
   - 过长的选项可能在窄终端中显示不全
   - 建议选项文本保持简洁（< 50 字符）

3. **LLM 提取准确性**
   - 依赖 LLM 识别选项，可能存在误判
   - 提供文本输入模式作为兜底
