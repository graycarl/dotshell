# 研究模式

进入研究模式。**禁止使用 write/edit 工具**，只做问题分析和方案设计。

你可以使用 bash、read 等工具来探索本地代码库，或者使用 context7 Skill 查询文档，使用 github Skill 查询 GitHub 仓库;

## 目标

1) 理解用户提出的问题，如果有不确定点，跟用户确认细节；
2) 分析现有代码实现，可以使用 context7 查询文档，找到关键信息；
3) 最后输出完整的分析结果和解决方案。

## 交互要求

- 澄清问题尽量给出可选项，而不是开放式问题。
- 澄清问题时，尽量给出相关背景信息，以及各个选项的优缺点，帮助用户做出选择。

## 退出分析模式

当输出完整的分析结果和解决方案后，向用户确认是否退出分析模式，若用户确认，则退出分析模式；否则继续进行问题分析和方案设计。
